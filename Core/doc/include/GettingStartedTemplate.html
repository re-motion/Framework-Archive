Rubicon .NET Commons

Getting Started

Rubicon.Data.DomainObjects

The Rubicon.Data.DomainObjects namespace and its sub namespaces contain interfaces and classes to be used for building domain models that are persisted by the DomainObjects persistence framework.

See the Rubicon.Data.DomainObjects namespace documentation for the starting points for working with the persistence framework.

Rubicon.ObjectBinding

The Rubicon.ObjectBinding namespace contains interfaces for implementing a business object model with a generic data definition and data access layer.

See the Rubicon.ObjectBinding namespace documentation for details on how to implement and use this layer.

Rubicon.ObjectBinding.Web

The Rubicon.ObjectBinding.Web namespace and its sub namespaces contain interfaces, classes, and web controls to be used for displaying and editing business objects in an ASP.NET environment.

See the Rubicon.ObjectBinding.Web namespace documentation for the starting points for working with the Rubicon.ObjectBinding.Web namespace.

Rubicon.Web.UI

The Rubicon.Web.UI namespace and its sub namespaces contain interfaces, classes, and web controls for implementing the user interface of an ASP.NET application.

See the Rubicon.Web.UI namespace documentation for the starting points for working with the Rubicon.Web.UI namespace.

The Rubicon.Web.UI namespace documentation also provides details on implementing accessible ASP.NET applications, as described in the Web Accessibility Initiative's (WAI) Web Content Accessibility Guidelines 1.0 (WCAG 1.0).

Rubicon.Web.ExecutionEngine

The Rubicon.Web.ExecutionEngine namespace contains interfaces and classes for implementing control flow logic in an ASP.NET application. The execution engine offers cross-page sequences and control flow features such as functions, parameters, and exception handling.

See the Rubicon.Web.ExecutionEngine namespace documentation for the starting points for working with the Rubicon.Web.ExecutionEngine namespace.
